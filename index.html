Quality Gates
5 camadas de validacao antes do commit
Nenhuma linha chega ao repositorio sem passar por validacao de sintaxe, seguranca, review por LLM, checagem de regressao e 14 passes de sanitizacao deterministica.
🔍

Validacao de Codigo

Syntax check (AST), instalacao de deps, resolucao de imports, pytest + coverage, deteccao de truncamento.

🔒

Scan de Seguranca

Bandit (vulnerabilidades Python) + Semgrep (padroes semanticos). Bloqueia commits inseguros.

🤖

Code Review por LLM

Qualidade, aderencia ao ticket, scope drift, over-engineering, risco de regressao.

📊

Regressao Incremental

Compara gerado vs existente. Bloqueia se >20% dos elementos (funcoes, classes, rotas) sumiram.

🛠

Sanitizacao (14 passes)

Bloqueio de arquivos perigosos, merge inteligente de compartilhados, auto-fix lint, correcao de imports.

Auto-Reparo
Duas camadas de recuperacao automatica
Erros sao classificados por tipo (syntax, import, test, security, truncation) e roteados para prompts especializados. Nada de "conserte tudo" generico.

Camada 1: Pre-Commit Barato

Resolve a maioria dos erros antes de tocar no repositorio.

  • FailureType enum classifica cada erro
  • Prompts especializados por tipo de falha
  • Ate 5 tentativas com deteccao de convergencia
  • Envia apenas arquivos afetados (economia de tokens)

Camada 2: Pos-CI Completo

Usa logs reais do GitHub Actions para corrigir falhas de integracao.

  • Aguarda CI completar via GitHub Checks API
  • Busca annotations + workflow logs
  • Ate 3 tentativas de correcao
  • Escala para review humano se nao converge

Transparencia
Onde precisamos melhorar
Engenharia de processo sofisticada rodando em infraestrutura artesanal. Estes sao os gaps reais que separam o RicardoDevin dos lideres do mercado.
⚠

Sem benchmark publico

Nao temos resultados no SWE-bench ou qualquer benchmark reproduzivel. Funciona nos nossos repos, mas sem evidencia de generalizacao.

⚠

Sem edicao cirurgica

Gera arquivos inteiros. Para mudar 1 linha, reescreve tudo. Em arquivos >500 linhas, risco real de truncamento e perda de conteudo.

⚠

Fire-and-forget

Sem como guiar o agente durante a execucao. Funciona de primeira (com retries) ou escala para review manual. Sem interacao no loop.

⚠

Modelos nao-frontier

DeepSeek v3 e Gemini Flash sao fortes, mas nao sao frontier para coding. Qualidade bruta de geracao inferior ao Claude Opus ou GPT-4o em tasks complexas.

⚠

Foco em Python

Frontend (React/TS) suportado com menos profundidade. Go, Rust, Java sem suporte em prompts nem validacao.

⚠

Single-threaded

1 issue por vez. Sem fila, sem monitoramento de saude, sem SLA. Sem comunidade ou ecossistema de plugins.

Veredicto: RicardoDevin tem engenharia de processo que compete conceitualmente com os melhores — reparo tipado, sanitizacao de 14 passes, e protecao de regressao vao alem do que a maioria oferece. Mas perde em potencia bruta: modelos nao-frontier, sem sandbox interativo, sem benchmark, sem escala, sem comunidade. E um sistema com engenharia de processo sofisticada rodando em infraestrutura artesanal. O caminho para competir de verdade passa por: benchmark publico (SWE-bench), sandbox com terminal, suporte multi-linguagem, e edicao cirurgica via diffs.
